Response to Gap Analysis & Technical Clarifications
Subject: Detailed Logic Definitions for DEFLUV SGC ERP

Thank you for the analysis. Below are the definitive answers to the questions raised and additional requirements for the system architecture.

0. REVISED USER HIERARCHY (Access Control)Implement three distinct tiers of access to prevent permission overlaps:TIER 1: ADMINAuthority: Absolute System Control.Permissions: Full CRUD on all modules, including "Control de Acceso" (User Management), "Registro Personal," and "Matriz de Documentos."Validation: The primary role authorized to change document states in the "Registro de Auditoría".TIER 2: SUB-ADMIN (Gestor de Tareas)Authority: Operational Departmental Management.Permissions: Full CRUD ONLY within the "Panel de Tareas" (Task Management). They can create, edit, delete, and approve/reject task files.Constraint: Functions as a "Regular User" in the Document Matrix and has no access to "Control de Acceso" or "Registro Personal."TIER 3: REGULAR USERAuthority: Contributor.Permissions: View assigned department only. Upload/replace files in the Matrix and Task Panel. No deletion or validation rights.1. Document Metadata & Workflow Dynamic Labels: Admin must define the document type (e.g., Plano, Factura) during request creation.Field "Artículo": Include a mandatory alphanumeric field called "Artículo" to link documents to specific MOP contract clauses.States Logic: Implement exactly 5 states for all document uploads:Pendiente: Initial state; awaiting review.Aprobado: Final positive state (Locked).Rechazado: Negative state; requires a mandatory "Rejection Comment" from Admin.Vencido: Automatic system flag if the deadline passes without an upload.No Cumple: Automatic state if a user uploads a file to a requirement that is already "Vencido" (Late Submission).2. Task Management (Panel de Tareas) Table Columns: Fecha de Ingreso, Solicitante, Tarea (Name/Title Only — NO description field), Responsable (Name + Dept), Estado (5-state logic), Plazo, and Acciones.File Logic: Each entry manages two files: the original instruction file (from Sub-Admin) and the resolution file (from User).Visibility: Users only see tasks for their assigned department.3. SGC-Bot (Gemini AI Integration) Scope: Use Gemini 1.5 to analyze PDFs against ISO-9001 and technical standards.Detail: Provide high-accuracy technical feedback. The bot's analysis is for advisory purposes and cannot change document states.4. Database Separation (Users vs. Personnel) System Users: Authenticated accounts with ERP access.Personnel Records: A separate HR database for field workers (drivers, operators) with no system login access.Bulk Import: Support Excel (.xlsx) for massive personnel data entry.5. NotificationsAdmin: Alerts on all file uploads/re-uploads.Users: Alerts on state changes, new document requests, or new task assignments.
