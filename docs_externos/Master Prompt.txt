## 3. USER ARCHITECTURE & RBAC (Role-Based Access Control)
- **ADMIN:** Full CRUD access. Manually registers users, assigns them to departments, manages the "Control de Acceso" matrix, and is the SOLE validator of documents.
- **REGULAR USER:** Assigned to 1 of 15 departments. Can ONLY see their department's tab. Can view protocols, upload documents, and replace files (with mandatory justification). Cannot delete.
- **15 DEPARTMENTS:** Bodega, Control de Gestión Administrativo de Obra, Administrativo Contable, Adquisiciones, Contabilidad, Control de Gestión, Laboratorio, Maquinaria, Personal, Prevención de Riesgos, Topografía, Gerencia de Administración y Finanzas, Gerencia de Operaciones, Gerencia General, Viviendas. (Admin can edit these).

## 4. FUNCTIONAL MODULES (As per UI Images)
1. **Panel de Control (Dashboard):** KPIs of compliance, active task panel, and "Upcoming Deadlines" (Tax payments: 2nd week of month; Salaries: before end of month).
2. **Matriz de Documentos:** A grid showing "Pending, Approved, Corregir, Rechazado, Vencido" status. Users upload files here; Admin reviews them.
3. **Registro Personal:** HR module for Admin/Personal staff. Features: Individual or Bulk upload (CSV/XLSX) of employee data, CVs, and certificates.
4. **Control de Acceso:** A visual matrix (refer to image image_f7dec0.png) where Admin toggles "Ver, Editar, Aprobar" permissions per department/user.
5. **Registro de Auditoría & AI Bot (SGC-Bot):** - A logs table with timestamps, user actions, and justifications.
   - **INTEGRATED AI (Gemini):** A side panel in the document viewer (refer to bot image) that reads the uploaded PDF and answers technical questions or checks for ISO-9001 compliance.

## 5. TECHNICAL REQUIREMENTS
- **Cloud Storage:** Files must be stored in a secure cloud bucket (e.g., Supabase Storage).
- **Audit Trail:** Every file replacement MUST trigger a log entry with a "Reason for change" comment.
- **File Preview:** Integrated PDF/Image viewer within the app to avoid unnecessary downloads.
